The Apache Air Control

An interactive guide to the world's most popular web server, explained through the real-life situation of managing an airport.

1. The Control Tower (The Server)

Think of a web server like Apache as the air traffic control tower for a busy airport. Its main job is to listen for incoming planes (requests from browsers) and tell them which terminal and gate (website files) to go to.

2. The Landing Process (Request/Response)

When you want to visit a website, your browser sends a plane (an HTTP Request) to the server's address. The Apache control tower receives this, finds the right gate, and sends a signal back (an HTTP Response) saying "OK, you can land here."

3. The Airport Map (Document Root)

The DocumentRoot is like the main map of your airport. It's the primary folder where Apache looks for all the gates (files). If a plane asks for /info, Apache looks for an info.html file inside this folder.

4. Multiple Airlines (Virtual Hosts)

One airport can serve many different airlines. Virtual Hosts allow one Apache server to manage multiple websites. The control tower looks at the airline on the plane's tail (the domain name) and directs it to the correct, dedicated terminal.
