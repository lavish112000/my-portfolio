2. Blueprints & Robots (Classes/Objects)

In Java, a Class is like a detailed blueprint for building a robot. It defines what parts it has (data) and what it can do (actions).

An Object is an actual robot built using that blueprint. You can create many robot objects from the same class blueprint.

Click the blueprint to build a new robot!

3. Robot Specs (Variables)

Variables are like the robot's specification sheet or memory. They store pieces of information, like its name (text, called a String), battery level (a number, like an int), or if its light is on (true/false, a boolean).

String robotName = "Bolt";
int batteryLevel = 100;
boolean lightOn = false;
Name: "Bolt"
Battery: 100%
Light: false

4. Robot Actions (Methods)

Methods are the actions or commands your robot can perform. They are defined within the class blueprint. When you want a specific robot object to do something, you "call" its method.

public void walk() {
  // Code for walking...
}

myRobot.walk(); // Call the method

5. Robot Decisions (If/Else)

Robots need to make decisions! Java uses if, else if, and else statements to check conditions. For example: "If the battery is low, find the charger, else continue the task."

if (batteryLevel < 20) {
  findCharger();
} else {
  continueTask();
}

6. Repeating Tasks (Loops)

Sometimes a robot needs to repeat an action, like taking 5 steps forward. Python uses loops (like for or while) to execute a block of code multiple times.

for (int i = 0; i < 5; i++) {
  robot.takeStep();
}
