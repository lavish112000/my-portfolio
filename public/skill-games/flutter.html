1. Everything is a Widget

The core philosophy of Flutter is that your entire UI is composed of widgets. Widgets are the building blocks of a Flutter app, describing what their view should look like given their current configuration and state. Your app is a tree of these widgets.

2. Stateless vs. Stateful

Widgets are either stateless or stateful. A StatelessWidget is immutable; it's drawn once with the information it's given and doesn't change. A StatefulWidget can change dynamically. Calling setState() tells Flutter that the widget's internal state has changed, causing it to be redrawn.

3. Layouts: Rows & Columns

Flutter makes layout simple with widgets like Row and Column. These flexible widgets let you arrange other widgets horizontally or vertically. You can control alignment and spacing with properties like mainAxisAlignment.

4. Hot Reload

One of Flutter's most powerful features is Hot Reload. It allows you to inject updated code into a running application almost instantly. You can fix bugs, build UI, and add features without restarting your app, preserving its current state.

5. One Codebase, Multiple Platforms

The ultimate goal of Flutter is to allow you to build and maintain one application that runs natively on multiple platforms. Your single Dart codebase compiles to ARM code for mobile, and to JavaScript for the web, providing true native performance everywhere.
