The Version Control Voyage

An interactive guide to the essential tools that power modern software development, helping you track changes and collaborate with teams.

1. A Time Machine for Your Work

Version Control is a system that records changes to a file or set of files over time so that you can recall specific versions later. It's like a "time machine" for your project.

Each time you save a "snapshot" of your project, it's called a commit. Add a commit message and save a snapshot, then click the points to travel back in time.

Version 1: The project begins.

2. The Three Areas of Git

Git doesn't just save files. It has a three-step process:
1. Working Directory: Your current files.
2. Staging Area: Files you've marked for the next snapshot. (git add)
3. Repository: Where the snapshot is permanently saved. (git commit)

Drag the file from Working to Staging, then click it to commit.

3. Branches: Parallel Universes

A branch is a parallel version of your project. It allows you to work on a new feature (e.g., "new-header") without affecting the main, stable version of your project (usually called main).

You can create branches, add commits to them, and they remain completely isolated until you're ready to combine them.

4. Merging: Combining Work

Once your feature is complete on its branch, you can merge it back into the main branch. This integrates the new feature into the main project, combining the histories of both branches.

This workflow allows teams to work on many features in parallel without getting in each other's way.

5. Remotes: Working with a Team

A remote repository (like on GitHub or GitLab) is a shared, central version of your project.
• git push: Send your local commits to the remote.
• git pull: Get the latest changes from the remote.

This is the foundation of collaboration with other developers.
