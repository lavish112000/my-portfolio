The Tailwind CSS Workshop

An interactive guide to the utility-first CSS framework for rapidly building modern, responsive user interfaces.

1. The Utility-First Workflow

Instead of writing custom CSS classes, you build designs by applying pre-existing, single-purpose utility classes directly in your HTML. This allows for rapid prototyping and development without ever leaving your markup.

Try it! Change the class names in the input below to style the button live.

2. Responsive By Design

Tailwind makes responsive design intuitive. Use breakpoint prefixes like md: or lg: to apply utilities only at specific screen sizes. This mobile-first approach makes complex layouts easy to manage.

<div class="flex flex-col md:flex-row">...</div>

3. Handling States

Style elements on different states like hover, focus, and more by adding state prefixes. For example, use hover:bg-cyan-700 to change the background color when a user hovers over an element.

<button class="... hover:bg-cyan-700 focus:ring-4">...</button>

4. Dark Mode Made Easy

Implementing a dark theme is trivial. Just add the dark: prefix to any utility you want to apply when dark mode is active. A simple class toggle on your `html` tag is all it takes to switch themes.

<div class="bg-white dark:bg-slate-800">...</div>

5. Just-In-Time Engine

Tailwind's JIT compiler generates your CSS on-demand by scanning your files for class names. This means your final CSS file is incredibly small, containing only the styles you actually use, ensuring lightning-fast load times.
