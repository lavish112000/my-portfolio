The Next.js Blueprint

A dynamic, interactive tutorial on the powerful React framework for building fast, modern, and full-stack web applications.

1. File-System Routing (Pages)

The traditional Next.js approach creates routes based on your file structure. Any React component in the pages directory automatically becomes a page.

  • pages/index.js → /
  • pages/about.js → /about

This intuitive system simplifies navigation. Click the files on the right to see it in action.

2. The App Router

The modern approach uses the app directory. Routing is folder-based, but you now create special layout.js and page.js files. This allows for more complex, nested layouts.

  • app/layout.js → Shared UI Shell
  • app/page.js → Content for /
  • app/dashboard/page.js → Content for /dashboard

3. Components

Next.js is built with reusable components. Create UI elements (like a Navbar) and import them into your pages to build complex interfaces without repeating code. Hover over the code tags to see the connection.

// app/page.js import Navbar from '../components/Navbar'; export default function HomePage() { return ( <> <Navbar /> <h1>Home</h1> <Footer /> </> ) }

4. Static Rendering

By default, the App Router renders components on the server statically. The HTML is generated at **build time** and cached on a CDN. This is perfect for pages that don't need real-time data, providing incredible speed and great SEO.

// This component is rendered at build time async function Page() { const res = await fetch('...'); const data = await res.json(); return <div>{data.title}</div>; }

5. Dynamic Rendering

For pages that need fresh data on every request, you can opt into dynamic rendering. The page is generated on the server for **each request**, ensuring the data is always up-to-date. This happens automatically when you use a dynamic function.

// This component is rendered on every request async function Page() { const res = await fetch('...', { cache: 'no-store' }); const data = await res.json(); return <div>{data.title}</div>; }

1. User Makes Request

Navigates to the page.

2. Page Rendered on Server

Data is fetched on-demand.

3. HTML Sent to User

The freshly generated page is returned.

6. Route Handlers (APIs)

You can create backend API endpoints using Route Handlers. A route.js file inside any app directory folder allows you to handle HTTP requests (GET, POST, etc.), enabling full-stack functionality like database access or form handling.
